simulink与MBD开发_C2000_Microcontroller_Blockset

概述

在matlab 2023a及其之后的版本集成C2000 Microcontroller Blockset，
可用于TI C2000系列MCU的MBD开发。

环境配置

在matlab命令行窗口，输入：

弹出配置窗口：

  • 勾选所要用到的处理器系列，点击下一步；
  • 对TI C2000 Code Generation Tools与TI ARM Code Generation Tools点击install进行安装，安装完成后点击下一步；
  • 配置窗口会对TI C2000开发三件套：
    • TI controlSUITE
    • TI Code Composer Studio
    • TI C2000Ware
      进行检测，如果未安装按照指示安装即可，（如果安装在非默认路径可以直接点击下一步；注意核实适配的版本，注意安装路径不能有中文），安装完成重启matlab重新输入命令打开配置窗口进行下一步即可；
  • 这一步需要验证三件套的安装位置，如果是默认安装位置点击Validate验证即可，如果是自定义路径则自己选择对应路径进行验证；
  • 最后会列出各工具链的版本，核实无误后即可完成。

demo

新建一个simulink模型，
此时在库浏览器中可以看到C2000 Microcontroller Blockset的库，里面包含了各种芯片的外设模块。

本次以驱动TMS320F28335的GPIO为例，拖出对应的GPIO模块Digital Output搭建一下模型：

配置模块为Toggle GPIO0，表示每执行一次该模块，当输入为1时，GPIO0输出状态翻转。

打开建模->模型设置：

  • 求解器
    • 停止时间设为inf
    • 求解器选择为：定步长、离散
    • 此时求解器采样步长即为处理器执行步长，
      这里设置为0.5，代表每0.5s执行一次;
  • 数学和数据类型
    • 将非规范数的仿真行为设为下溢为零(FTZ)。
    • 由于C2000系列最大支持32位，因此数据类型默认设定为single（即float型）
      • 一些芯片不支持浮点型，因此实际使用时注意类型转换
  • 硬件实现
    • Hardware board选择为对应的芯片，对于F28335，选择TI Delfino F2833x，等待自动配置完成；
    • 点开Hardware board settings->Target hardware resources，配置编译选项(Build options)：
      • build action选择为build,load and run
      • 勾选Use custom linker command file，设定linker command file(cmd文件)和CCS hardware configuration file(ccxml文件)。
    • Clocking：根据实际器件配置时钟。
    • 如果使用到其它外设，在这里可以相应进行设置。
  • 代码生成
    • 系统目标文件选择ert.tlc（嵌入式系统）
    • 工具链选择Texas Instruments Code Composer Studio(C2000)
      • 在下方高级参数(默认是三个点)可以对工具链进行验证。
    • 代码生成目标配置为执行优先、ROM优先、RAM优先
    • 优化
      • 如果需要阅读代码，则将默认参数行为设为可调，优化级别设定为最小值
    • 报告：勾选创建代码生成报告、生成模型web视图

配置完成后，上方工具栏会新增一个HARDWARE,将设备通过烧录器连接到电脑（注意上电顺序），点击编译、部署与启动，即生成固件并烧录进设备里，正常情况下可以观测到GPIO0端口输出每0.5ms翻转一次。

simulink进行在环调试

上位机与处理器之间可通过串口进行通信，从而实现对处理器的监控与调整。
对于C2000系列处理器，只支持SCI_A串口。

在硬件方面，可以基于CH340G的USB转TTL串口模块将上位机与处理器进行连接，
要特别注意处理器的上电顺序，避免损坏设备。

如果干扰不严重可以只连接TX与RX两根线，因为有些PCB板设计不合理，串口模块的电源与地会直接给处理器供电，造成上电顺序错误。

对于F28335，其SCI_A可以用GPIO28 <-> GPIO29、GPIO35 <-> GPIO36两种组合，simulink默认使用第一种（当然，可以更改）。

如果是第一次使用串口模块，上位机需要安装对应的驱动，
并在上位机的设备管理器中设置串口数据帧的格式。
该数据帧格式与在simulink中设置的相匹配即可。

打开建模->设置->硬件实现->Hardware board settings->Target hardware resources：

  • 配置SCI_A
    • 默认波特率使用115200，这些设置与电脑的设备管理器中设置中相同即可。
  • 配置External mode：
    • XCP on Serial
    • SCI_A
    • 电脑对应的端口(可点击refresh进行检测)
    • Verbose与set logging buffer size automatically可以都勾上
    • 对于传输数据的准确率有高要求，可以勾选使用专用定时器

回到主页面，点击上方工具栏的HARDWARE，
选择Run on board(External mode),
点击“监控并调节”就能自动进行编译、烧录，并建立上位机对设备的监控。

SIL与PIL

参考链接

在 SIL 或 PIL 仿真中，代码是为顶层模型或模型的一部分生成的。
对于 SIL，此代码针对计算机编译并在其上执行；
对于 PIL，代码针对目标硬件进行交叉编译，并在目标处理器上运行。

通过通信信道，Simulink 在仿真的每个采样间隔向计算机（SIL）或目标处理器（PIL）发送激励信号。

对于顶层模型，Simulink 使用来自基础工作区或模型工作区的激励信号。

如果指定仅模型的一部分在 SIL 或 PIL 模式下进行仿真，则模型的其余部分将保留在 Simulink 中，并且不会为模型的此保留部分生成代码。
模型的非代码生成部分可以表示算法的其他部分或算法运行的环境。

当计算机或目标处理器接收到来自 Simulink 的信号时，处理器执行 SIL 或 PIL 算法，执行时间为一个采样步长。
SIL 或 PIL 算法通过通信信道将在此步长内计算的输出信号返回到 Simulink。
仿真的一个采样周期完成后，Simulink 进入下一个采样间隔。此过程不断重复，仿真持续进行。
SIL 和 PIL 仿真不以实时方式运行。
在每个采样周期内，Simulink 和目标代码会交换 I/O 数据。

要注意无论是SIL还是PIL，算法代码都不是实时地运行，
要实现实时运行仿真，可以采用HIL，但需要购置昂贵的IO服务器用以实时模拟被控对象。

SIL的配置与PIL的配置很像，先对PIL进行说明。
以下是基于搭建的子系统创建PIL模块，
该PIL模块可以直接放到模型中进行仿真运行。

在上面配置上位机与硬件通讯的基础上，
在建模->设置->代码生成->验证中，勾选启用可移植字长，打开高级参数（下方三个点），创建模块设为PIL。

在模型仿真中，一般模块的执行步长设定为继承，再配合Rate Transition模块即可对执行步长进行配置，
此时模型的固定步长小于等于所有模块的执行步长即可；
但在生成PIL模块之前，一定要将模型的固定步长设定为代码所需的执行步长，待生成完成后再改回原来所需的步长即可。

设定好步长后，
右键点击所需编译的子系统/模块，
选择C/C++代码->部署此子系统到硬件，即可自动编译、生成PIL模块。

注意生成时matlab文件夹要处于当前模型处，
生成代码时会自动生成Subsystem_ert_rtw的子文件夹，里面是各种代码与目标文件。
要注意编译生成的子系统代码时本级路径是模型所在文件夹\Subsystem_ert_rtw，比模型的本级路径多一级，
因此子系统中的S-Function模块使用了相对路径时，要注意这个路径问题，
可以把使用的源文件复制一份到对应的路径。

在APP中选择SIL/PIL Manager可以对仿真行为与监控进行设置。

对于SIL模块，基本配置都是相同的，只需在
建模->设置->代码生成->验证->高级参数（下方三个点）中，创建模块`设为SIL即可。
