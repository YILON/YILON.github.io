Flash芯片AT45D161D

发表于2024-02-24|更新于2024-03-18|嵌入式IC
|浏览量:

参考自官方数据手册。

概述

AT45DB161D是一款2.5V或2.7V，串行接口的FLASH存储器。
AT45DB161D支持RapidS串行接口，适用于要求高速操作的应用。
RapidS串行接口兼容SPI，最高频率可达66MHz。
AT45DB161D的存储容量为16Mbit。

除了主存储器外，AT45DB161D还包含2个512/528字节的SRAM缓冲区。
Buffer允许在对主存储器的页重新写入时接收数据，也可写入连续的数据串。
通过独立的“读-改-写”三步操作，可以轻松实现EEPROM仿真（可设置成位或字节）。
DataFlash通过RapidS串行接口顺序访问数据，而不像传统FLASH存储器那样通过复用总线和并行接口随机存取。

简单顺序访问机制极大的减少了有效引脚的数量，有利于硬件布局，增强了系统可靠性，将切换噪音降至最小，并缩小了封装的尺寸。

为了实现简单的在系统重复写入，AT45DB161D并不需要高输入电压来支持写入。
AT45DB161D工作在独立的2.5V至3.6V或者2.7V至3.6V电压下，用于写入和读取操作。

    • 代码映射应用的理想选择

  • 硬件和软件数据保护特性

  • 数据保存20年 工业级温度范围

封装

主要引脚

输入引脚：

  • 片选引脚CS‾\mathrm{\overline{CS}}CS
    • 低电平有效。（下降沿表示通信开始，上升沿表示通信结束）
  • 时钟引脚SCK\mathrm{SCK}SCK
  • 输入引脚SI\mathrm{SI}SI
  • 输出引脚SO\mathrm{SO}SO
    • 当未通讯时Flash处于standby mode，输出引脚SO\mathrm{SO}SO处于高阻抗。
  • 写保护引脚WP‾\mathrm{\overline{WP}}WP
    • 当该引脚低电平时，禁止写入或擦除Flash。
    • 不要让该引脚悬空。
  • 复位引脚RESET‾\mathrm{\overline{RESET}}RESET
    • 当该引脚低电平时，会终止正在进行的操作，并将内部状态机置为idle状态(空闲状态)。
    • 不要让该引脚悬空。

输出引脚：

  • 就绪/繁忙引脚RDY/BUSY‾\mathrm{RDY/\overline{BUSY}}RDY/BUSY
    • 当器件忙于内部自定时操作时，这个漏极开路输出引脚将被拉低。
      该引脚通常处于高电平状态（通过外部上拉电阻器），
      在写入/擦除操作、比较操作和页到Buffer传输期间将被拉低。
    • 忙状态表示无法访问闪存阵列和其中一个Buffer；
      但仍可对另一个Buffer执行读写操作。

存储结构

由上可知，
可以直接读取Flash的页(main page)，但不能直接写入，
而是要先通过缓冲区才能写入。
对于缓冲区，可以直接读取与写入。

AT45DB161D 的存储器阵列分为三个级别：

  • 扇区SECTOR：
    • 总共有16扇区，其中第0个扇区分为0a与0b两个子扇区；
    • 一个扇区有256页，即32个块。
  • 块BLOCK：
    • 一个块有8个页；
    • 总共有512个块；
  • 页PAGE：
    • 一个页大小可以为标准页大小528bit或512bit；
    • 整块芯片总共有4096个页。

对数据闪存的所有写入操作都是逐页进行；
读取则可以按字节进行；
擦除操作可在芯片、扇区、块或页级执行。

地址说明

由上可知，

  • 主存储器字节寻址共22bit，
    • 其中 PA11 - PA0 表示指定页地址所需的12个地址位，
    • BA9 - BA0 表示指定页内字节地址所需的10个地址位。
      对于标准 DataFlash页大小（528 字节），
  • 缓冲区寻址则使用 BFA9 - BFA0 表示指定缓冲区内字节地址所需的10个地址位；

对于二进制页大小（512 字节），

  • 主存储器字节寻址共使用21bit，
    • 其中 A20 - A9 表示指定页地址所需的12个地址位，
    • A8 - A0 表示指定页内字节地址所需的9个地址位。
  • 缓冲区寻址则使用 BFA8 - BFA0 来表示缓冲区内指定字节地址所需的9个地址位；

根据层次扇区-块-段，对主存储器，

扇区地址需要前4bit，
块地址需要前9bit，
页地址则需要前12bit。

Flash基本操作

所有指令、地址与数据都以MSB格式传输。

读操作

基于SPI协议，通讯都是从CS‾\mathrm{\overline{CS}}CS下降沿开始。

连续阵列读取 E8H(传统指令，最高66MHz)

给定一个主存储器的起始字节地址，便一直读下去，直到片选信号拉高。
连续阵列读取绕过两个数据缓冲区，并保持缓冲区的内容不变。

开始时主机先向Flash发送命令与要读取的地址：

  • OPCODE即命令(8bit)，

  • ADDRESS BITS即要读取的主存储器起始字节地址
    由于以8 bit一单位发送数据，则地址位需要凑够为8的整数

    • 若是标准页大小，字节地址为22位，因此在前面添加两个无意义位凑够24位的地址。
    • 若是二进制页大小，字节地址为21位，因此在前面添加三个无意义位凑够24位的地址。

  • DON’T CARE BITS是无意义位（为了满足时序需求）

    • 对于两种页大小的阵列读取，地址后Don't care码长度都为32 bit。

随后Flash开始向主机返回对应的数据，
只要CS‾\mathrm{\overline{CS}}CS就一直返回。

此模式读取下如果要跨页，将在下一页开始时继续读取，
在页边界交叉（从一页结束到下一页结束的交叉）期间不会产生延迟。

当主存储器阵列中的最后一位已被读取时，
Flash将继续在存储器的第一页的开始处读回。
与跨越页边界一样，从数组的末尾到数组的开头换行时不会产生延迟。

连续阵列读取 0BH(高频指令，最高66MHz)

给定一个起始地址，便一直读下去，直到片选信号拉高。
连续阵列读取绕过两个数据缓冲区，并保持缓冲区的内容不变。

    • 若是二进制页大小，字节地址为21位，因此在前面添加三个无意义位凑够24位的地址。

  • DON’T CARE BITS是无意义位（为了满足时序需求）

    • 对于两种页大小的读取，地址后Don't care码为8 bit。

只要CS‾\mathrm{\overline{CS}}CS就一直返回。

在页边界交叉（从一页结束到下一页结束的交叉）期间不会产生延迟。

连续阵列读取 03H(低频指令，最高33MHz)

给定一个起始地址，便一直读下去，直到片选信号拉高。
连续阵列读取绕过两个数据缓冲区，并保持缓冲区的内容不变。

开始时主机先向Flash发送命令与要读取的地址：

  • OPCODE即命令(8bit)，
  • ADDRESS BITS即要读取的起始地址
    由于以8 bit一单位发送数据，则地址位需要凑够为8的整数

    • 若是标准页大小，字节地址为22位，因此在前面添加两个无意义位凑够24位的地址。

    • 若是二进制页大小，字节地址为21位，因此在前面添加三个无意义位凑够24位的地址。

    • 地址后无Don't care码。

随后Flash开始向主机返回对应的数据，
只要CS‾\mathrm{\overline{CS}}CS就一直往下读。

此模式读取下如果要跨页，将在下一页开始时继续读取，
在页边界交叉（从一页结束到下一页结束的交叉）期间不会产生延迟。

主存储器页读取 D2H

直接从主存储器中4096个页中的任何一个字节读取数据，
绕过两个数据缓冲器并保持缓冲器的内容不变。

只要片选引脚CS‾\mathrm{\overline{CS}}CS不拉高就一直往下读，
当到达主存储器中指定页的末尾时，Flash将继续从同一页的开头读回。

  • OPCODE即命令(8bit)，

  • ADDRESS BITS即要读取的起始地址
    由于以8 bit一单位发送数据，则地址位需要凑够为8的整数

    • 若是标准页大小，字节地址为22位，因此在前面添加两个无意义位凑够24位的地址。
    • 若是二进制页大小，字节地址为21位，因此在前面添加三个无意义位凑够24位的地址。

  • 对于两种页大小的读取，地址后Don't care码长度都为32 bit。

缓冲区读取 D4H、D1H、D6H、D3H

SRAM数据缓冲器可以独立于主存储器阵列进行访问。
Buffer 1使用D4H（低频，最高66MHz）与D1H（低频，最高33MHz），
Buffer 2使用D6H（低频，最高66MHz）与D3H（低频，最高33MHz）。
只要片选引脚CS‾\mathrm{\overline{CS}}CS不拉高就一直往下读，
当到达缓冲区的末尾时，Flash将继续从缓冲区开头读回。

  • OPCODE即命令(8bit)，

  • ADDRESS BITS即要读取的起始地址

    • 若是标准页大小，此时该地址（缓冲区地址）为10 bit。
    • 若是二进制页大小，此时该地址（缓冲区地址）为9 bit。

  • 对于标准页大小的缓冲区读取，地址前Don't care码为14 bit。

  • 对于二进制页大小的缓冲区读取，地址前的Don't care码为15 bit。

特别地，在低频读取时(D4H 或 D6H)，
地址后还有8 bitDon't care码。

主存储器读到缓冲区 53H、55H

Buffer 1使用53H
Buffer 2使用55H

在执行过程中，状态寄存器和RDY/BUSY‾\mathrm{RDY/\overline{BUSY}}RDY/BUSY指示正忙。

对于标准页大小，地址前的Don't care码为2 bit。
对于二进制页大小的擦除，地址前的Don't care码为3 bit。

  • 两种页的地址都为12个地址位。

对于标准页大小的擦除，地址后的Don't care码为10 bit。
对于二进制页大小的擦除，地址后的Don't care码为9 bit。

写操作

缓冲区写入 84H、87H

  • 84H用于写入Buffer 1，
  • 87H用于写入Buffer 2。

对于标准页大小的缓冲区写入，地址前的Don't care码为14 bit。
对于二进制页大小的缓冲区写入，地址前的Don't care码为15 bit。

  • 若是标准页大小，此时地址（缓冲区地址）为10 bit。
  • 若是二进制页大小，此时地址（缓冲区地址）为9 bit。

当到达缓冲区的末尾时，Flash将继续从缓冲区开头写入。

带内置擦除功能的缓冲区到主存储器 83H、86H

此功能会先将主存储器指定页擦除，然后将缓冲区数据写入该页。
在执行过程中，状态寄存器和RDY/BUSY‾\mathrm{RDY/\overline{BUSY}}RDY/BUSY指示正忙。
执行时间不长于40ms，典型时间为17ms。

  • 83H用于Buffer 1写入主存储器，
  • 86H用于Buffer 2写入主存储器。

对于标准页大小的写入，地址前的Don't care码为2 bit。
对于二进制页大小的写入，地址前的Don't care码为3 bit。

  • 若是标准页大小，此时地址为12个页面地址位（PA11-PA0）。
  • 若是二进制页大小，此时地址为12个页面地址位（PA11-PA0）。

对于标准页大小的写入，地址后的Don't care码为10 bit。
对于二进制页大小的写入，地址后的Don't care码为9 bit。

不带内置擦除功能的缓冲区到主存储器 88H、89H

必须先发送对应指令将主存储器指定页擦除，
然后才能使用该指令将缓冲区数据写入该页。
在执行过程中，状态寄存器和RDY/BUSY‾\mathrm{RDY/\overline{BUSY}}RDY/BUSY指示正忙。
缓冲区写入主存储器执行时间不长于6ms，典型时间为3ms。

对于标准页大小的写入，地址前的Don't care码为2 bit。
对于二进制页大小的写入，地址前的Don't care码为3 bit。

  • 若是标准页大小，此时地址为12个页面地址位（PA11-PA0）。
  • 若是二进制页大小，此时地址为12个页面地址位（PA11-PA0）。

对于标准页大小的写入，地址后的Don't care码为10 bit。
对于二进制页大小的写入，地址后的Don't care码为9 bit。

  • 83H用于Buffer 1写入主存储器，
  • 86H用于Buffer 2写入主存储器。

对于标准页大小的写入，地址前的Don't care码为2 bit。
对于二进制页大小的写入，地址前的Don't care码为3 bit。

  • 若是标准页大小，此时地址为12个页面地址位（PA11-PA0）。
  • 若是二进制页大小，此时地址为12个页面地址位（PA11-PA0）。

对于标准页大小的写入，地址后的Don't care码为10 bit。
对于二进制页大小的写入，地址后的Don't care码为9 bit。

通过缓存写入主存储器 82H、85H

即执行
数据写入缓冲区、
擦除主存储器指定页并将缓冲区数据写入、
二者结合起来。
在缓冲区写入主存储器过程中，状态寄存器和RDY/BUSY‾\mathrm{RDY/\overline{BUSY}}RDY/BUSY指示正忙。
缓冲区写入主存储器执行时间不长于17ms，典型时间为40ms。

  • 82H用于写入Buffer 1，
  • 85H用于写入Buffer 2。

对于标准页大小的写入，地址前的Don't care码为2 bit。
对于二进制页大小的写入，地址前的Don't care码为3 bit。

  • 若是标准页大小，此时地址为

    • 12个页面地址位（PA11-PA0）
    • + 写入缓冲区字节地址(10 bit)

  • 若是二进制页大小，此时地址为

    • 12个页面地址位（PA11-PA0）
    • + 写入缓冲区字节地址(9 bit)

两种页大小地址后都没有Don't care码

跟在地址后的是要写入的数据。

当CS‾\mathrm{\overline{CS}}CS引脚上升沿后（写入缓冲区完成），
该部件将首先将主存储器中的所选页面擦除为全部1，
然后将存储在缓冲器中的数据写到该存储器页面中。

当写入到达缓冲区的末尾时，Flash将继续从缓冲区开头写入。

自动页面重写(APR) 58H、59H

一般情况下，
往主存储器写入一页数据需要先往缓冲区写入，
再让缓冲区数据写入主存储器。

但实际上这只适合按顺序写入，即依次逐页地写入。

如果需要频繁地随机写入某页，
这种做法不仅耗时长，而且可能会使其它页的数据发生错误。
（这个和NOR flash性质有关）

该芯片手册要求扇区中的每个页，
必须在该扇区中每10000次累积页面擦除/编程操作中至少更新/重写一次，
以确保数据不发生错误。

APR就是将主存储器指定页的数据写到缓冲区中，然后再写回去。
在执行过程中，状态寄存器和RDY/BUSY‾\mathrm{RDY/\overline{BUSY}}RDY/BUSY指示正忙。

  • 58H用于Buffer 1，
  • 89H用于Buffer 2。

对于标准页大小的写入，地址前的Don't care码为2 bit。
对于二进制页大小的写入，地址前的Don't care码为3 bit。

  • 两种页大小对应的地址都是12 bit

对于标准页大小的写入，地址后的Don't care码为10 bit。
对于二进制页大小的写入，地址后的Don't care码为9 bit。

擦除

主存储器页擦除 81H

在执行过程中，状态寄存器和RDY/BUSY‾\mathrm{RDY/\overline{BUSY}}RDY/BUSY指示正忙。
擦除页执行时间不长于35ms，典型时间为15ms。

对于标准页大小的擦除，地址前的Don't care码为2 bit。
对于二进制页大小的擦除，地址前的Don't care码为3 bit。

  • 两种页大小的地址皆为12个地址位（PA11-PA0）。

对于标准页大小的擦除，地址后的Don't care码为10 bit。
对于二进制页大小的擦除，地址后的Don't care码为9 bit。

块擦除 50H

一块的大小是8页。

在执行过程中，状态寄存器和RDY/BUSY‾\mathrm{RDY/\overline{BUSY}}RDY/BUSY指示正忙。
擦除块执行时间不长于100ms，典型时间为45ms。

对于标准页大小的擦除，地址前的Don't care码为2 bit。
对于二进制页大小的擦除，地址前的Don't care码为3 bit。

  • 两种块地址都为9个地址位。

对于标准页大小的擦除，地址后的Don't care码为13 bit。
对于二进制页大小的擦除，地址后的Don't care码为12 bit。

扇区擦除 7CH

一个扇区的大小是256页。

在执行过程中，状态寄存器和RDY/BUSY‾\mathrm{RDY/\overline{BUSY}}RDY/BUSY指示正忙。
擦除块执行时间不长于5s，典型时间为1.6s。

对于标准页大小的擦除，

  • 如果是0a或0b扇区

    • 地址前的Don't care码为2 bit。
    • 地址为9个页面地址位。
    • 地址后的Don't care码为13 bit。

  • 如果是01~15扇区

    • 地址前的Don't care码为2 bit。
    • 地址为4个扇区地址位。
    • 地址后的Don't care码为18 bit。

对于二进制页大小的擦除，

  • 如果是0a或0b扇区

    • 地址前的Don't care码为3 bit。
    • 地址为9个页面地址位。
    • 地址后的Don't care码为12 bit。

  • 如果是01~15扇区

    • 地址前的Don't care码为3 bit。
    • 地址为4个扇区地址位。
    • 地址后的Don't care码为17 bit。

整芯擦除 C7H_94H_80H_9AH

执行芯片擦除命令，命令序列为4字节：C7H、94H、80H和9AH。
由于要擦除整个存储器阵列，因此不需要将地址字节，
并且在操作码之后计时的任何数据都将被忽略。

当CS‾\mathrm{\overline{CS}}CS上升沿后开始擦除，
在执行过程中，状态寄存器和RDY/BUSY‾\mathrm{RDY/\overline{BUSY}}RDY/BUSY指示正忙。
片擦除命令不会影响受保护或锁定的扇区。

WP‾\mathrm{\overline{WP}}WP引脚可以在设备擦除时被置低，
但在内部擦除周期完成之前，保护不会被激活。

Flash操作的说明

Flash操作可分为四类。

A 组：

  1. 主存储器页读取
  2. 连续阵列读取
  3. 读取扇区保护寄存器
  4. 读取扇区锁定寄存器
  5. 读取安全寄存器

B 组包括：

  1. 页擦除
  2. 块擦除
  3. 扇区擦除
  4. 芯片擦除
  5. 主内存页到Buffer 1（或 2）传输
  6. 主存储器页到Buffer 1（或 2）比较
  7. Buffer 1（或 2）至主存储器页写入（带内置擦除）
  8. Buffer 1（或 2）至主存储器页程序（无内置擦除）
  9. 通过Buffer 1（或 2）进行主存储器页写入
  10. 自动页重写

C 组包括：

  1. Buffer 1（或 2）读
  2. Buffer 1（或 2）写
  3. 状态寄存器读
  4. 制造商和Flash ID 读取

D 组包括：

  1. 擦除扇区保护寄存器
  2. 对扇区保护寄存器写入
  3. 扇区锁定
  4. 写入安全寄存器

如果 A 组操作正在执行中（未全部完成），则不应启动 A、B、C 或 D 组中的另一个操作。
在 B 组操作的内部自定时部分，可以执行 C 组中的任何操作。
使用Buffer 1 的 B 组命令应使用Buffer 2 的 C 组操作，反之亦然。
最后，在 D 组操作的内部自定时部分，只能执行状态寄存器读取操作。

操作对应命令

读操作：

  • 主存储器页读取 D2H
  • 连续阵列读取（传统命令） E8H
  • 连续阵列读取（低频） 03H
  • 连续阵列读取（高频） 0BH
  • Buffer 1 读取（低频） D1H
  • Buffer 2 读取（低频） D3H
  • Buffer 1 读取 D4H
  • Buffer 2 读取 D6H

写操作：

  • Buffer 1 写入 84H
  • Buffer 2 写入 87H
  • Buffer 1 至有内置擦除功能的主存储器页程序 83H
  • Buffer 2 至有内置擦除功能的主存储器页程序 86H
  • Buffer 1 至无内置擦除功能的主存储器页程序 88H
  • Buffer 2 至无内置擦除功能的主存储器页程序 89H
  • 页擦除 81H
  • 块擦除 50H
  • 扇区擦除 7CH
  • 芯片擦除 C7H、94H、80H、9AH
  • 通过Buffer 1 执行主存储器页程序 82H
  • 通过Buffer 2 执行主存储器页程序 85H

保护与安全命令：

  • 启用扇区保护 3DH + 2AH + 7FH + A9H
  • 禁用扇区保护 3DH + 2AH + 7FH + 9AH
  • 擦除扇区保护寄存器 3DH + 2AH + 7FH + CFH
  • 写入扇区保护寄存器 3DH + 2AH + 7FH + FCH
  • 读取扇区保护寄存器 32H
  • 扇区锁定 3DH + 2AH + 7FH + 30H
  • 读取扇区锁定寄存器 35H
  • 写入安全寄存器 9BH + 00H + 00H + 00H
  • 读取安全寄存器 77H

额外命令：

  • 主存储器页到Buffer 1 传输 53H
  • 主存储器页到Buffer 2 传输 55H
  • 主存储器页到Buffer 1 比较 60H
  • 主存储器页到Buffer 2 比较 61H
  • 通过Buffer 1 自动重写页 58H
  • 通过Buffer 2 自动重写页 59H
  • Deep Power-down B9H
  • 从深度掉电恢复 ABH
  • 状态寄存器读取 D7H
  • 制造商和Flash ID 读取 9FH
