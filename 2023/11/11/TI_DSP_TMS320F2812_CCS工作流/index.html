默认会将文件放在C\tidcs\c28\DSP281x\v120中。
主要关注这几个文件夹：

  • DSP281x_common：主要放置例程要使用的源文件
  • DSP281x_examples：主要放置例程
  • DSP281x_headers：主要放置DSP281x_common对应的头文件

在CCS中新建工程

对于非BIOS项目，
通常选择"Empty Project"的“Empty Project(with main.c)”

新建完成后，配置一下，使得目录结构为：

  • TargetConfigs（放置cmd文件）
  • Debug（创建工程时自动生成，放置Debug版本的编译相关文件）
  • Lib
    • include
    • source
  • User
    • header
    • src

添加必要的头文件与配置工程

将"\DSP281x_headers\include"中的所有头文件复制到"\Lib\include"目录；
将"\DSP281x_common\include"中的所有头文件复制到刚刚新建的include目录（会提示文件已存在，覆盖即可）。
在CCS中右键点击工程文件夹，点击的"Properties"，
在"C2000 Compiler->include Options"中将对应的头文件路径添加到工程中，
在"C2000 Linker->Basic Options"中设置C System Stack Size（默认为空，编译时会发警告，可设置比如为0x400）
如果有引入外部库，在"C2000 Linker->File Search Path"中将添加外部库的路径；

在"C2000 Compiler->Advance Options -> Language Options"中可更改使用的C/C++标准（默认C标准为C89，可更改为C99）

添加必要的源文件

  • 添加"DSP281x_headers\source"目录下的DSP281x_GlobalVariableDefs.c以指定所有系统结构体的存储位置。
  • 添加"DSP281x_common\source"目录下的DSP281x_SysCtrl.c以初始化系统的控制选项，包括PLL，看门狗，预分频。
  • 添加"DSP281x_common\source"目录下的DSP281x_Gpio.c以初始化系统的GPIO口。
  • 添加"DSP281x_common\source"目录下的DSP281x_PieCtrl.c以初始化PIE控制器。
  • 添加"DSP281x_common\source"目录下的DSP281x_PieVect.c以初始化PIE向量表。
  • 添加"DSP281x_common\source"目录下的DSP281x_InitPeripherals.c以初始化其他外设。
  • 添加"DSP281x_common\source"目录下的DSP281x_DefaultIsr.c以使用默认的中断函数。

添加cmd文件

将新建工程时自动生成的cmd文件删除，
如果项目是非BIOS的，添加"DSP281x_headers\cmd"目录下的DSP281x_Headers_nonBIOS.cmd，
否则添加DSP281x_Headers_BIOS.cmd。

在Debug模式下，希望装载到RAM中，
因此此时添加"DSP281x_common\cmd"目录下的F2812_EzDSP_RAM_lnk.cmd；

在Release模式下（即确定了固定到Flash中的程序），
添加"DSP281x_common\cmd"目录下的"F2812.cmd"。记得将之前的F2812_EzDSP_RAM_lnk.cmd删除或者右键"Exclude"。
同时添加"DSP281x_common\source"目录下的DSP281x_CodeStartBranch.asm以重定向cpuBOOT后跳转的代码执行地址入口。

系统初始化

一般在main函数中都要执行这些：

将部分程序放到RAM中运行

一方面，
当程序烧写进Flash时，默认的速度是很慢的（等待时间很长），
因此要对Flash进行初始化配置，
但这些初始化指令本身存放在Flash中，
因此需要先将这些指令先加载到RAM中。

另一方面，
代码从内部flash里运行比从内部RAM里运行要慢30%左右，
故而，需要将代码拷贝到RAM以提高运行速度。

为了实现上面的两个目的，
定义这里的一个段ramfuncs。

  • 添加"DSP281x_common\source"目录下的DSP281x_MemCopy.c.

在F2812.cmd中，有这样一段内容：

  • 第1行表示该段的装载在PAGA0的FLASHD中，
  • 第2行表示该段的运行地址在PAGE0的RAML0中，
  • 第3行令编译器创建了一个变量_RamfuncsLoadStart，
    该变量指向段ramfuncs的装载地址的首地址（LOAD_ START为编译伪指令，见CCS的帮助文档）；
  • 第4行编译器创建了一个变量_RamfuncsLoadEnd，
    该变量指向段ramfuncs的装载地址的末地址（LOAD_ END为编译伪指令，见CCS的帮助文档）；
  • 第五行令编译器创建了一个变量_RamfuncsRunStart，
    该变量指向段ramfuncs的运行地址的首地址（RUN_ START为编译伪指令，见CCS的帮助文档）。
