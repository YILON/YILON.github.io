TI_DSP_TMS320F2812_存储空间与启动过程

发表于2023-11-11|更新于2023-11-30|嵌入式DSP
|阅读量:

F28xx映射空间概述

F28xx采用增强的哈佛总线结构，能并行访问程序和数据，
32位数据地址，22位程序地址，最大可寻址4G字（每字16位）的数据空间和4M的程序空间。
采用统一寻址方式（程序、数据、IO统一寻址），地址映射如下：

  1. 片上SRAM
    （1）M0和M1；每块大小为1Kx16bit，其中M0映射到0x000000-0x0003FF，M1映射到0x000400-0x0007FF。
    （2）L0和L1；每块大小为4Kx16bit，其中L0映射到0x008000-0x008FFF，L1映射到0x009000-0x009FFF。
    （3）H0；大小为8Kx16bit，映射到0x3F8000-0x3F9FFF。
  2. 片上Flash
    F2812芯片内部包含一片128Kx16bit的Flash存储器（代码可直接在上面运行），映射到0x3D8000-0x3F8000。
  3. 片上OTP
    F2812芯片内部有一块1Kx16bit的一次性可编程存储器（One-Time-Programmable），映射到0x3D7800-0x3D7BFF。
  4. 片上BootROM
    F2812芯片内部有一块4Kx16bit的BootROM（引导ROM），映射到0x3FF000-0x3FFFFF。
    在存储器内TI公司装载了产品版本号、发布的数据、校验求和信息、复位向量、CPU向量表及标准的数学运算表（如正弦表、余弦表）等。
    BootROM的主要作用是实现DSP的BootLoader功能，
    芯片出厂时在BootROM的0x3FFC00-0x3FFFBF空间内有厂家的引导装载程序，当芯片被设置为为控制器模式时，CPU在复位后将执行这段程序。

在软件编译链接过程中，
需要通过.cmd或DSP/BIOS指定代码段、数据段、堆栈等对象的映射空间。

XINTF接口扩展

存储器接口负责将CPU访问存储器逻辑控制单元同存储器、外设以及其他的接口连接起来。
存储器接口包含独立的数据和程序总线，
因此cpu能在一个周期内能够同时访问程序存储器和数据存储器。
该接口还包含存储器访问需要的各种控制信号(如读写等)，
通过这些信号控制存储器或外设的数据传输。
除了16位和32位格式的数据访问外，F2812还支持特殊的字节访问指令，
通过这些特殊的字节访问指令可以分别访问一个字的高字节(MSB)和低字节(LSB)。

当访问相应的存储空间时，会产生一个片选信号；
其中有的存储空间共用一个片选信号，使用时采用统一的编址方式。
每个空间都可以独立地设置访问等待、选择、建立以及保持时间，
同时还可以使用XREADY信号来控制外设的访问。

外部接口的访问时钟频率由内部的XTIMCLK提供，
XTIMCLK可以等于SYSCLKOUT或 SYSCLKOUT/2。

在复位状态下，根据XMP/MC‾\rm{XMP/\overline{MC}}XMP/MC的状态,处理器选择微处理器或微计算机操作模式。
在微处理器模式(XMP/MC‾=1\rm{XMP/\overline{MC}}=1XMP/MC=1)下，Zone7映射到高位置地址空间，
中断向量表可以定位在外部存储空间。
在这种工作模式下BootROM将被屏蔽。

在微计算机模式(XMP/MC‾=0\rm{XMP/\overline{MC}}=0XMP/MC=0)下，Zone7被屏蔽且从Boot ROM获取中断向量表。
因此用户可以选择从片上存储器或片外存储器启动应用程序。

上电复位时，XMP/MC‾\rm{XMP/\overline{MC}}XMP/MC的状态存放在XINTCNF2寄存器的XMP/MC‾\rm{XMP/\overline{MC}}XMP/MC模式位，
用户也可以通过软件改变该位控制 Boot ROM和XINTF Zone 7的映射。
其他存器并不受XMP/MC‾\rm{XMP/\overline{MC}}XMP/MC状态的影响。
注意，F2812的外部扩展接口并不支持I/O空间。

XINTF接口操作

在F2812上有些空间共用同一个片选信号，如：
Zone0和Zone1共用XZCS0ANDCS1‾\rm{\overline{XZCS0ANDCS1}}XZCS0ANDCS1；
Zone6和Zone7共用XZCS6ANDCS7‾\rm{\overline{XZCS6ANDCS7}}XZCS6ANDCS7。
各空间均可以独立设置访问等待、选择、建立以及保持时间。
所有空间共享19位的外部地址总线，处理器根据所访问的空间产生相应的地址。

Zone2和Zone6

Zone2和Zone6共享外部地址总线，
范围为0x00000~0xFFFFF。

访问Zone2和Zone6的唯一区别是片选信号的不同，
分别是XZCS2‾\rm{\overline{XZCS2}}XZCS2与XZCS6ANDCS7‾\rm{\overline{XZCS6ANDCS7}}XZCS6ANDCS7，
因此二者地址分别为
0x08 0000 ~ 0x09 FFFF;
0x10 0000 ~ 0x17 FFFF.

Zone0和Zone1

Zone0和Zone1共用XZCS0ANDCS1‾\rm{\overline{XZCS0ANDCS1}}XZCS0ANDCS1,
但内部地址不同。
Zone0的寻址范围为0x20000~0x3FFFF，
Zone1的寻址范围为0x40000~0x5FFFF。

因此要区分这两个空间，

从而实现两个空间的映射地址范围为
0x00 2000 ~ 0x00 3FFF;
0x00 4000 ~ 0x00 5FFF.

Zone7

Zone7是一个独立的地址空间，
复位时，如果XMP/MC‾\rm{XMP/\overline{MC}}XMP/MC引脚为高电平，
Zone7空间映射到0x3FC000。

系统复位后，可以通过改变寄存器XINTCNF2中的XMP/MC‾\rm{XMP/\overline{MC}}XMP/MC控制位，
使能或屏蔽Zone7空间。
如果XMP/MC‾\rm{XMP/\overline{MC}}XMP/MC引脚为低电平，
则Zone7不能映射到0x3FC000上，而是由片上的ROM映射到此处空间。

如果需要建立自己的引导程序并存放在外部空间，
可以使用Zone7空间进行程序的引导。
引导成功后，通过软件使能内部的ROM，
以便访问放在ROM中的数学表。

Boot ROM映射到Zone7时，原来Zone7对应的存储器空间仍可以通过Zone6访问。
这是因为二者使用同一个片选线。

Flash

在F2812中，
包含128K×16位的Flash，
其又可以被分成可单独擦除、编程、验证的4个8K×16位的段(Sector)和6个16K×16位的段。

Flash特点：

  • 可分多个Sector；
  • 代码安全保护机制CSM；
  • 低功耗模式；
  • 可根据CPU频率调整等待周期；
  • 采用专用的流水线操作

F2812的Flash地址分配

C28x启动顺序

通常将DeBug版本的可执行代码装载到内部RAM中（比如H0-SRAM）方便调试，加快速度，
而对于release版本的固件则写入到非易失性存储器中，
系统每次上电后能够采用特定的引导操作自动运行。

C28x上电后有6种不同的启动模式，

注意，无论采取哪种启动模式，
程序最终只能从Flash、OTP或H0-SRAM中执行程序。

从Flash开始运行

要使cpu从内部的Flash开始运行，
除了拉高F4引脚，
还需要将cpu配置位计算机模式（XMP/MC‾=0\rm{XMP/\overline{MC}}=0XMP/MC=0）。

  1. 系统复位后程序指针跳转到BROM（BOOT ROM）的起始地址0x3F FFC0；
  2. 执行BROM中的跳转指令到引导代码首地址0x3F FC00，完成基本的初始化任务与引导模式的选择；
  3. 如果F4引脚置高，PC指针直接跳转到0x3F 7FF6（Flash入口地址）。
    该段只有2个字长，在这两个字中写入跳转到应用程序入口的跳转指令等，
    分别是跳转指令LB、应用程序入口地址、CSM的密码
    比如使用C开发，则跳转到c_int00函数。
  4. cint00函数完成C环境和全局变量的初始化。该段必须放在Flash中。
  5. 执行完cint00函数后，调用主程序main。

其它启动方式有需要再查看教材。

BOOT ROM基本情况介绍

该段包含了一些常用的数据表（比如三角函数）用以通过查表的方式快速运算，
还有BootLoader。

BootLoader


第一个字作为引导程序的特征值，用来确定引导程序采用8位还是16位宽的数据进行装载。
比如对于SPI引导，只支持8位的引导装载，此时该值设位0x08AA，
否则设为0x10AA。

如果BootLoader接收到无效的特征字会退出装载程序，
系统从Flash执行。

第2~9个字用来初始化寄存器的值，或用来改进BootLoader操作，也可以不使用。
在使用SPI模式引导时会使用一个字初始化SPI寄存器。

第10~11个字构成22位程序入口地址，在Bootloader完成后使用该值初始化PC指针。

数据流中的第12个字表示第一个程序数据块的大小，
无论采用8位还是16位的数据格式该值都以16位定义。
例如传输32个8位程序数据，块的大小则定义为0x0010，表示16个16位字的程序数据。

接下来的2个字定义程序数据块的目的地址。

每个数据块都按上述格式组织(数据块大小目的地址)，
系统在引导装载过程中重复传输。
一旦遇到需要传输的数据块的大小等于0就表示所有数据块传输完成。
然后转载程序将返回到人口地址调用程序，
由BootLoader中确定的程序人口地址开始执行应用程序。

Flash的初始化

cpu访问Flash需要等待一段时间。
对于F281x处理器，退出复位时默认访问Flash需要16个等待状态，
这无疑带来巨大的延迟，大大降低处理速度。

因此需要根据实际情况配置合适的等待状态数量。

C环境下如果工作在150MHz，要求最少加入5个等待。
但此时执行单周期指令的频率为150/(5+1)=25MHz，
仍然浪费了很多等待时间。

因此实际上还采用了流水线的方式访问Flash（将ENPIPE控制位置1），
每(1+5)个周期访问4条指令，此时执行指令的速度为100MHz。
（在系统上电时Flash流水操作是禁止的）

关于CMD文件

转载链接
CCS通过CMD文件指定存储区域的分配。
F2812的CMD采用的是分页制，
PAGE0用于存放程序空间，
PAGE1用于存放数据空间。

MEMORY和SECTIONS

参考链接
二者是命令文件中最常用的两伪指令。
MEMORY伪指令用来表示实际存在目标系统中的可以使用的存储器范围，
在这里每个存储器都有自己的名字，起始地址和长度。

SECTIONS伪指令是用来描述输入端是如何组合到输出端内的。

CMD文件里有两个基本的段：初始化段和非初始化段。
初始化段包含代码和常数等必须在DSP上电之后有效的量。
故初始化块必须保存在如片内FLASH等非遗失性存储器中。

非初始化段中含有在程序运行过程中才有的变量，
所以非初始化段必须链接到易失性存储器中如RAM。

初始化段：
.text：所有可以执行的代码和常量
.cinit:全局变量和静态变量的C初始化记录
.const：包含字符串常量和初始化的全局变量和静态变量（由const）的初始化 和说明
.econst：包含字符串常量和初始化的全局变量和静态变量（由far const）的初 始化和说明
.pinit：全局构造器（C++）程序列表
.switch：包含switch声明的列表

非初始化段：
.bss: 为全局变量和局部变量保留的空间，在程序上电时.cinit空间中的数据复制 出来并存储在.bss空间中。
.ebss：为使用大寄存器模式时的全局变量和静态变量预留的空间，在程序上电 时，cinit空间中的数据复制出来并存储在.ebss中
.stack:为系统堆栈保留的空间，用于和函数传递变量或为局部变量分配空间。
.sysmem:为动态存储分配保留的空间。如果有宏函数，此空间被宏函数占用，如果没有的话，此空间保留为0
.esysmem:为动态存储分配保留的空间。如果有far函数，此空间被相应的占用， 如果没有，此空间保留为0.
