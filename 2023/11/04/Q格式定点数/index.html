Q格式定点数

发表于2023-11-04|更新于2024-08-26|C语言
|阅读量:

引用链接

概述

Q格式是二进制的定点数格式。
相对于浮点数，Q格式指定了相应的小数位数和整数位数。
在没有浮点运算的平台上，可以更快地对浮点数据进行处理，
以及应用在需要恒定分辨率的程序中（浮点数的精度是会变化的）。

Q数据的表示

记定点数格式为Qm,nQ_{m,n}Qm,n​，
mmm为整数位数，nnn为小数位数，
对于有符号数，最高位为符号位并用补码形式存储。

假设某16位定点数XXX在Q9,7Q_{9,7}Q9,7​格式下为：

X=1011011000011101X = 1011 \quad 0110 \quad 0001 \quad 1101 X=1011011000011101

如果X是无符号数

即整数部分为：

101101101101101101 101101101

小数部分为：

00111010011101 0011101

则整数部分：

X1=1×28+0×27+1×26+1×25+0×24+1×23+1×22+0×21+1×20=365X_1 = 1 \times 2^{8} + 0 \times 2^{7} + 1 \times 2^{6} + 1 \times 2^{5} + 0 \times 2^{4} + 1 \times 2^{3} + 1 \times 2^{2} + 0 \times 2^{1} + 1 \times 2^{0} = 365 X1​=1×28+0×27+1×26+1×25+0×24+1×23+1×22+0×21+1×20=365

小数部分：

X2=0×2−1+0×2−2+1×2−3+1×2−4+1×2−5+0×2−6+1×2−7=0.2265625X_2 = 0 \times 2^{-1} + 0 \times 2^{-2} + 1 \times 2^{-3} + 1 \times 2^{-4} + 1 \times 2^{-5} + 0 \times 2^{-6} + 1 \times 2^{-7}=0.2265625 X2​=0×2−1+0×2−2+1×2−3+1×2−4+1×2−5+0×2−6+1×2−7=0.2265625

所以：

X=X1+X2=365.2265625X = X_1 + X_2 = 365.2265625 X=X1​+X2​=365.2265625

如果X是有符号数

则取补（按位取反末尾再加1）得到真值：

Xreal=0100100111100011X_{real} = 0 \quad 10010011 \quad 1100011 Xreal​=0100100111100011

可见为一个正数，

Xreal1=1×27+0×26+0×25+1×24+0×23+0×22+1×21+1×20=147X_{real_1} = 1 \times 2^{7} + 0 \times 2^{6} + 0 \times 2^{5} + 1 \times 2^{4} + 0 \times 2^{3} + 0 \times 2^{2} + 1 \times 2^{1} + 1 \times 2^{0} = 147 Xreal1​​=1×27+0×26+0×25+1×24+0×23+0×22+1×21+1×20=147

Xreal2=1×2−1+1×2−2+0×2−3+0×2−4+0×2−5+1×2−6+1×2−7=0.7734375X_{real_2} = 1 \times 2^{-1} + 1 \times 2^{-2} + 0 \times 2^{-3} + 0 \times 2^{-4} + 0 \times 2^{-5} + 1 \times 2^{-6} + 1 \times 2^{-7}=0.7734375 Xreal2​​=1×2−1+1×2−2+0×2−3+0×2−4+0×2−5+1×2−6+1×2−7=0.7734375

所以此时

X=Xreal1+Xreal2=147.7734375X = X_{real_1} + X_{real_2} = 147.7734375 X=Xreal1​​+Xreal2​​=147.7734375

范围与精度

对有符号数：

  • 范围：[−2m−1,2m−1−2−n][-2^{m-1},\quad 2^{m-1}-2^{-n}][−2m−1,2m−1−2−n]
  • 精度：2−n2^{-n}2−n

对无符号数：

  • 范围：[0,2m−2−n][0,\quad 2^{m}-2^{-n}][0,2m−2−n]
  • 精度：2−n2^{-n}2−n

Q格式定点数的舍入

以IQ22的定点数为例，其格式为：

【第31位符号位】【30 ~ 22位整数位】【21~0位小数位】

Q格式定点数本质上是以long格式（32 bit）存储的。

而对于整型变量，

更长的数据赋值给短数据时，长度超出的部分舍弃，

比如32位赋值给16位，

长数据舍去第30~第15位（从0开始计算），

第31的符号位赋予短数据的第15位，

剩下的一一赋值。

对于IQ22个数的定点数，当只要求保留整数，则右移22位即可。

要注意的是，对于IQ15~IQ1格式的定点数，

其整数部分已经超出16位能表示的最大值(-65536~65535)，

这种情况下必须丢弃整数若干位的精度，且其大小会被缩放。

这种情况一般发生在通讯的过程中，

比如数据以 16bit 的长度下位机发送给上位机，

此时便可以先将数据进行缩放，

传输到上位机后再乘回去。

如果传输需要在确定整数最大位数的前提下保留小数位以尽可能保留精度，
