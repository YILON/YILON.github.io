TI_DSP_TMS320F2812_SPI接口

发表于2023-11-04|更新于2023-11-23|嵌入式DSP
|阅读量:

概述


SPI总线上只能有一个主机(Master)，
其它都是从机(Slave)。
一般地，可有几种工作情况：

  • Master发送数据，Slave发送伪数据；
  • Master发送数据，其中一个Slave发送数据；（全双工）
  • Master发送伪数据，其中一个Slave发送数据。

C28x的SPI模块概述

C28x有以下特点：

  • 4个外部引脚
    • SPISOMI\rm{SPISOMI}SPISOMI：从输出/主输入引脚
    • SPISIMO\rm{SPISIMO}SPISIMO：从输入/主输出引脚
    • SPISTE‾\rm{\overline{SPISTE}}SPISTE：从发送使能引脚
    • SPICLK\rm{SPICLK}SPICLK：时钟引脚
  • 波特率：125种可编程波特率
  • 数据字长：可编程1~16个数据长度
  • 4种时钟模式（这里说明SPI通讯方式中的另一种解释形式，和上面说的奇\偶数边沿采样本质一样）
    • 无相位延时的下降沿；SPICLK 为高电平有效。在 SPICLK 信号的下降沿发 送数据，在 SPICLK 信号的上升沿接收数据。
    • 有相位延时的下降沿：SPICLK 为高电平有效。在 SPICLK 信号的下降沿之 前的半个周期发送数据，在 SPICLK 信号的下降沿接收数据。
    • 无相位延时的上升沿：SPICLK 为低电平有效。在 SPICLK 信号的上升沿发 送数据，在SPICLK 信号的下降沿接收数据。
    • 有相位延时的上升沿：SPICLK 为低电平有效。在 SPICLK 信号的下降沿之 前的半个周期发送数据，而在 SPICLK 信号的上升沿接收数据。
  • 两种缓冲模式
    • 标准SPI模式
      • 接收操作使用双缓冲（发送操作并不支持）
        即在新的接受操作启动时，
        cpu可以暂时不读取SPIRXBUF中接收到的数据
        但在新的接收操作完成之前必须读取SPIRXBUF
        否则将会覆盖原来接收到的数据
    • FIFO缓冲模式
      • 用户可以建立16级深度的发送和接收缓冲堆栈
        而程序操作仍然使用SPITXBUF和SPIRXBUF寄存器
        这样使得SPI模块具有接收/发送16次数据的能力
        此模式下还可以根据两个FIFO的数据装载状态确定其中断级别
  • 增强特点：
    • 16级发送/接收FIFO
    • 延时发送控制

主机模式(MASTER/SLAVE = 1)

此时DSP的SPI模块为SPI总线提供时钟。
通过SPI波特率寄存器SPIBRR可设置126种数据传输速率。

主机通过SPISIMO引脚来发送数据，而通过SPISOMI引脚输入数据。
当数据写到移位寄存器SPIDAT或者写到串行发送缓冲器SPITXBUF的时候，
会启动SPISIMO引脚开始发送数据，
首先发送的是SPIDAT的最高位MSB，接着将剩余的数据左移1位，
然后将接收到得数据通过SPISOMI引脚移入SPIDAT的最低有效位LSB。

当指定数量的数据位通过移位寄存器SPIDAT移位后，发生以下事件：

  • 移位寄存器SPIDAT中的内容传递到输入缓冲寄存器SPIRXBUF中
  • SPI状态寄存器SPISTS的中断标志位SPI INT FLAG(SPISTS.6)置位。
    • 如果SPI操作控制寄存器SPICTL的SPI中断使能位SPI INT ENA置位，则产生中断请求。
  • 如果在发送缓冲寄存器SPITXBUF中还有有效的数据（由SPISTS寄存器中的TXBUF FULL位指示），则将指定字长的数据装载进移位寄存器SPIDAT，继续进行数据传输，否则立即停止SPICLK时钟。

从机模式(MASTER/SLAVE = 0)

此时SPI总线时钟由总线中的主机提供，DSP作为从机。
注意SPICLK引脚输入频率不应超过DSP的LSPCLK频率的1/4倍。

此模式下数据传输方式也是类似的。

SPI操作控制寄存器SPICTL的TALK位置零会禁止数据发送，
输出引脚SPISOMI变为高阻状态。
如果在发送数据期间将该位置零，会等到数据传输完再将输出引脚置为高阻。

当对应的SPISTE‾\rm{\overline{SPISTE}}SPISTE引脚为高电平时，
从设备的引脚也会被置为高阻状态，
从而保证同一时刻内SPISOMI总线只会被一个从设备占用。

FIFO操作

系统上电复位时，SPI缓冲模式为标准SPI模式，禁止FIFO功能，
FIFO的寄存器SPIFFTX、SPIFFRX和SPIFFFCT不起作用。

可通过将FIFO发送控制寄存器SPIFFTX的SPIFFENA置位使能FIFO模式。
可通过该寄存器的SPIRST位再操作任一阶段复位FIFO模式。

发送和接收缓冲器使用2个16×16 FIFO。
对于标准SPI模式的一个字长的发送缓冲器SPITXBUF，
在FIFO缓冲模式中作为发送FIFO与移位寄存器SPIDAT之间的发送缓冲器，
即FIFO先将数据装载到SPITXBUF中，
SPITXBUF再把数据装载进SPIDAT中。

FIFO中的字装载到SPIDAT之间可设置一定时间的延时。
0延时的话FIFO的数据一位紧接一位放到SPIDAT中发送，
而最大可设置256个SPI时钟周期的延时，
即字与字装载之间间隔256个SPI时钟周期。
这种功能使得f2812的SPI模块能适配速率较慢的外设。

FIFO模式有2个中断，
一个用于发送FIFO、SPITXINT，
另一个用于接收FIFO、SPIINT/SPIRXINT。

对于SPI FIFO接收来说，
产生接收错误或者接收FIFO溢出都会产生SPINT/SPIRXINT。

SPI波特率设置寄存器SPIBRR

在主机模式下，
该寄存器可以设定SPI时钟的波特率：

  • 当SPIBRR = 0~2时：

    BaudRate=LSPCLK4\rm{Baud Rate = \frac{LSPCLK}{4}} BaudRate=4LSPCLK​

  • 当SPIBRR = 3~127时：

    BaudRate=LSPCLKSPIBRR+1\rm{Baud Rate = \frac{LSPCLK}{SPIBRR+1}} BaudRate=SPIBRR+1LSPCLK​

可见总共可设计125种波特率。

在从机模式下，该寄存器无效。
注意输入SPI频率不能大于LSPCLK4\rm{\frac{LSPCLK}{4}}4LSPCLK​。
