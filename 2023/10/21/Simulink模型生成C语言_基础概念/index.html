Simulink模型生成C语言_基础概念

发表于2023-10-21|更新于2023-10-22|MATLABSimulinkEmbedded Code
|阅读量:

参考链接

基本概念

MATLAB/Simulink中一共提供三个代码生成的工具：

  • MATLAB Coder——用于m脚本生成C/C++代码
  • Simulink Coder——用于模型、Stateflow等生成C/C++代码
  • Embedded Coder——用于模型生成嵌入式代码

要调用模型生成的C语言函数，则模型中要定义接口：

  • 输入接口——即input port模块
  • 输出接口——即output port模块

同时模型要满足：

  • 模型是离散的（即各模块都是离散的）
  • 求解器为定步长(Fixed-step)

代码生成设置

点击上方栏：建模(MODELING)->模型设置(Model Settings) ->代码生成(Code Generation)

系统目标文件选择erl.tlc便可使用Embedded Coder生成嵌入式C语言；
编译过程中如果勾选"仅生成代码"与"代码和工件打包"可生成相关文件的压缩包(.zip):

如果需要模块的参数以变量形式生成代码，在“优化”中将默认参数行为改为可调(tunable)

对于接口选项，根据实际输入信号的类型选择，确实不需要支持的类型可以不勾选，以节省空间：

代码生成

在“APP”中打开Embedded Coder，点击“Generate Code”即可生成C代码。
(不管是否生成代码压缩包，都会生成"模型名_ert_rtw"的文件夹，里面也有同样的c文件，也有生成过程中产生的一些中间文件、报告等)

C代码的文件结构

一般生成的压缩包中会有这些文件：

  • ert_main.c （代码调用例程）
  • 模型名.c (模型算法初始化、实现、结束)
  • 模型名.h （模型相关数据的定义）
  • 模型名_private.h
  • 模型名_types.h （一些变量的重命名）
  • rtwtypes.h （相关数据类型的声明）
  • 模型名_data.c （模块参数变量赋值）

模型调用方式

在“模型名.c”中定义了函数 void 模型名_step(void)，
每调用一次该函数，相当于让模型进行一步的运算。
（可以发现该函数默认没有输入输出，因为模型的输入输出被定义为了全局变量）
一般将该函数放入定时器中断服务函数中，实现固定频率的离散计算。

信号

即模型中不同模块之间的信号传递线。

外部信号便是指与输入输出端口连接的信号线(如图中的PI_Ref、PI_Fbk、PI_Out)，
默认生成的结构体变量是全局变量；

内部信号便是指其它有分叉点的信号线(如图中的Err)，
默认产生局部变量，随着step函数出栈而被释放。
没有分叉点时的信号(如Vp、Vi)不会生成变量,而是隐含在算法过程中。

对于重要的信号线，
最好还是为其命名，以方便在代码中识别出来。

参数

参数指的便是各模块中可设定的参数（比如PID模块的kp、ki、kd）。
如果“代码生成->优化”那里没有将默认参数行为改为可调(tunable)，则模块参数将以常数的形式呈现在代码中，
反之，则会将各模块的参数定义在一个结构体变量中。

因此，对于重要的模块参数，要为其模块命好名以便查看。

状态

上面知道，
在每一个离散的时间点， 运行一次Step函数。（又称为一步）
运行一次Step函数，
除了需要输入数据以外，（为输入信号结构体变量赋值）
往往还需要上一个时刻的运算结果，甚至之前连续几个时刻的运算结果。

这些结果便用状态变量保存。（全局变量）

在Simulink模型库中，凡是包含离散因子“z”的模块，全部具有状态变量。
这些模块在生成代码时，都会生成一个名为“模型名_DW”的变量来保存状态变量。

在Simulink中，
通过Data Store Memory模块可实现创建自己的“状态变量”，
这些状态变量同样放在 模型名_DW中。

模型数据

模型数据结构体存放了模型的部分信息。
默认情况下，模型数据只包含了一个表示错误状态的字符串。

其它说明

在“模型名.c”中，
除了 模型名_step() 函数外，
还有初始化函数与终止函数，
前者和使用外设一样要对模型进行初始化，
后者默认情况下是空的，可以不用管。

在ert_main.c 中展示了对step函数的一种封装形式，将Step函数进行了一层封装形成rt_OneStep()函数，添加了错误状态检查功能，可以参考。

在“代码生成->优化”中，
有“指定自定义优化(Specify custom optimizations)”选项，
默认是关闭的，打开后可自定义优化选项。

其中“信号存储复用（Signal store reuse）”是比较重要的选项，
如果取消勾选，则其会定义各模块的输出信号（即内部信号中无分叉点的信号线也会被定义），
统一在结构体“模型名_B”中定义。
这样可以提升代码的可阅读性，但是降低了效率与空间利用率。
