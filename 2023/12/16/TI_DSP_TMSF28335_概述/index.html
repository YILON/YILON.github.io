工程搭建

使用C2000 WARE套件中的例程进行基本工程搭建。

安装C2000 WARE套件后，在C:\ti\c2000\C2000Ware_x_xx_xx_xx\device_support\f2833x（默认安装路径）中能找到F2833x的官方例程。

新建一个文件夹作为基本工程文件夹，
在其中新建若干文件夹，形成如此结构：

  • Project folder
    • lib
      • header：用以放置官方例程中基本配置与外设的头文件
      • src：用以放置官方例程中基本配置与外设的源文件
    • include：放置用户创建的头文件
    • source：放置用户创建的源文件
    • bsp：放置开发板的外设驱动文件(由开发板厂家提供)
  • 将官方例程的\common\include与\headers\include中所有头文件，复制到工程文件夹的\lib\header，
  • 将官方例程中的\common\cmd\F28335.cmd、\common\cmd\28335_RAM_Ink.cmd、\headers\cmd\DSP2833x_Headers_nonBIOS.cmd添加到工程根目录；
    （如果使用BIOS则将DSP2833x_Headers_nonBIOS.cmd替换为DSP2833x_Headers_BIOS.cmd）
  • 将官方例程的\headers\source的所有源文件放到工程文件夹的\lib\src中；
  • 将官方例程的common\source所有源文件放到工程文件夹的\lib\src中；
    （除了DSP2833x_SWPrioritizedDefaultIsr.c与DSP2833x_SWPrioritizedPieVect.c，否则会造成冲突，等到需要用时再具体阐述）
  • 将厂家提供的开发板外设驱动文件放到工程文件夹的\bsp中。

这些文件的作用在前面的CCS工作流中有介绍。

打开CCS，创建空白工程（有条件最好接上仿真器，这样配置好仿真器类型时可以点击verify进行校验），删除默认生成的28335_RAM_Ink.cmd。
将上面的工程文件夹添加到工程中（最好是复制到工程中）。
右键点击工程，选择最后一项Properties：

  • 在Build/C2000 Compiler/include Options中添加工程中存在头文件的路径与库的路径；
  • 在General/project type and tool-chain中配置编译器版本（在创建工程时也可以配置），
    如果所需编译器版本不存在，需要额外安装（安装路径为\ti\ccxxxx\ccs\tools\compiler），
    在学习期间本人使用的版本是v6.4.2
  • 在General/project type and tool-chain中配置Linker command file为DSP2833x_Headers_nonBIOS.cmd
  • 关于烧写
    • 如果要烧写在RAM中，在工程中将F28335.cmd右键exclude掉；
    • 如果要烧写在FLASH中，则在工程中将28335_RAM_Ink.cmd右键exclude掉，其余两个cmd文件则包含进来；
