概述

德州仪器 TMS320C28x IQmath 库是一套高度优化的高精度数学函数库，
供C/C++程序员在TMS320C28x器件上将浮点算法无缝移植到定点代码中。
这些例程通常用于计算密集型实时应用，在这些应用中，最佳执行速度和高精度至关重要。
通过使用这些例程，可以获得比用标准ANSIC语言编写的同等代码快得多的执行速度。

IQmath库包含五部分：

  • IQmath头文件
    • C 程序使用IQmathLib.h
    • C++ 程序使用IQmathLib.h和IQmathCPP.h
  • IQmath库
    该库包含所有 IQmath 函数和查找表。该库有两个版本:
    • IQmath.lib:
      这是一个索引库，根据 CCS 10.x 中的选项，项目中将使用相应的 EABI 或 COFF 库。
      该索引库使用IQmath_coff.lib和IQmath_eabi.lib库创建。
    • IQmath_f32.lib:
      这是一个索引库，根据 CCS 10.x 中的选项，项目中将使用相应的 EABI 或 COFF 库。
      该库可与使用 --float_support=fpu32 构建的代码链接。
      该索引库使用 IQmath_fpu32_coff.lib 和 IQmath_fpu32_eabi.lib 库创建。
    • IQmath_coff.lib：
      这是 IQmath 库的 COFF 版本。该库专为在C28x上运行。构建该库时未使用fpu_support = fpu32。
    • IQmath_eabi.lib：
      这是IQmath库的EABI版本。该库专为在C28x上运行。构建该库时未使用-fpu_support = fpu32。
    • IQmath_fpu32_coff.lib：
      这是IQmath库的 COFF 版本。该库用于在C28x+FPU设备上运行。该库使用 -fpu_support = fpu32 构建。
    • IQmath_fpu32_eabi.lib：
      这是IQmath库的EABI版本。该库用于在C28x+FPU设备上运行。该库使用 -fpu_support = fpu32 构建。
  • IQmath.cmd
    链该文件分配了 IQmath 库使用的部分.
    dsp中某些段的位置由设备指定.
    例如，IQsin 和 IQcos 功能使用的表格位于设备的Boot ROM 中。
  • GEL 文件
    这些gel函数在使用不直接支持 IQ 数据类型的 Code Composer Studio 版本时非常有用。
    使用 Code Composer Studio V3.3 或 Code Composer Studio V4 时不需要 .gel 文件。
  • examples

安装

IQmath库包含在Ti公司提供的ControlSUITE套件。

controlSUITE 不再更新，但仍可继续下载。
C2000的最新软件版本现在通过 C2000Ware 提供，并通过 DigitalPower SDK 和 MotorControl SDK 提供应用示例。

安装C2000ware后，
默认在C:\ti\c2000Ware_X_XX_XX_XX\libraries\math\IQmath\c28处为IQmath库。

将 IQmath 运算转换为浮点运算

  • 在IQmath头文件启用FLOAT_MATH，这将转换所有IQmath函数为等效的浮点型函数。
  • 将浮点数写入设备寄存器时，需要将浮点数转换为整数。
    同样，从寄存器读取数值时，也需要将其转换为浮点数。
    在这两种情况下，都需要将数值乘以一个转换系数。
    例如，浮点型数据转换为IQ15，需要×32768.0。
  • 如果设备具有片上浮点处理单元（C28x+FPU），则可以通过以下操作利用：
    • 使用v5.0.2以上版本的C28x codegen tools
    • 告诉编译器可以生成本地 C28x 浮点代码。
      为此，使用 -v28–float_support=fpu32编译器选项。
      • In Code Composer Studio V4:
        C/C++ build options->Tools Settings Tab -> Runtime Model Operations:
      • In Code Composer Studio V3.3:
        the float_support switch is on the advanced tab of the compiler options.
    • 为本地 32 位浮点运算使用正确的运行时支持库。C 代码使用 rts2800_fpu32.lib。
      对于带有异常处理功能的 C++ 代码，使用 rts2800_fpu32_eh.lib。
    • 使用 C28x FPU 快速 RTS 库 (SPRC664) 可提高 sin、cos、div、sqrt 和 atan 等数学函数的性能。
      快速 RTS 库应在normal run-time support库之前链接。

c程序与c++程序使用IQmath的不同点

C程序使用IQmathLib.h；
C++ 程序使用IQmathLib.h和IQmathCPP.h。

C程序使用的相关宏与函数名前有下划线_，而C++没有；
C程序的Q格式数基本运算需要使用专门的函数，而C++进行了运算符重载，可直接使用基本运算符；

IQmath数据段和查找表

一些IQmath函数使用查找表。
c28x设备的BootROM中包含了许多这样的查找表。
使用BootROM副本可以节省内存空间和加载表本身所需的时间。
因为一些设备上的BootROM为1等待状态，而SARAM通常为0等待状态。
查找表的访问频率通常不足以导致性能问题。
因此，在大多数情况下，你会希望使用Boot ROM查找表。

IQmath库使用这些查找表：

  • IQmathTables：
    本数据段包含最常用 IQmath 函数的查找表。
    这包括 lQdiv、IQsin、IQcos、IQsqrt 和 IQatan2。
    所有 28x 设备的Boot ROM 中都有 IQmathTables。
    因此，应在链接器命令文件中将该部分标识为 "NOLOAD "类型。
    这样就可以方便地引用查找表符号，而无需将该部分实际加载到目标中。
  • IQmathTablesRam:
    本节包含 IQexp、IQasin 和 IQacos 函数使用的初始化查找表。
    如果不调用这些函数，则无需加载此部分。
    如下表所示，某些设备还在Boot ROM 中已包含了其中一些表。

IQmathTablesRam 部分的名称可能有点误导，但由于历史原因一直保留下来。
该表包含 IQexp、IQasin 和 IQacos 函数的初始化数据表。
在调试过程中，这些表可以直接加载到 SARAM 中。
在实际运行期间，如果要使用这些函数，则应将这些表加载到非易失性存储器（如flash）中。
如果要在 SARAM 中访问这些表，则应在初始化过程中将其从闪存复制到 SARAM。

如果设备的Boot ROM 中不包含相关表，
则必须在cmd文件中将其加载到相应的内存中。
