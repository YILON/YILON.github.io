Simulink_基于simscape模型生成HDL代码

发表于2025-10-24|更新于2026-02-05|MATLABSimulink
|浏览量:

概述

硬件在环仿真HIL，或者俗称半实物仿真，指的是使用实际物理控制器，而被控对象则由相应HIL设备实时模拟，
对于控制器而言，和在物理实际运行别无二致。

要实现对被控对象的实时模拟，
基本原理便是基于控制理论，得到被控对象的离散状态方程，
转译为执行代码，在处理器中运行。

一般地，离散状态方程可如此描述：

x(k+1)=Ax(k)+Bu(k)y(k)=Cx(k)+Du(k)\begin{align*} \mathbf{x}(k+1) &= \mathbf{A}\mathbf{x}(k) + \mathbf{B}\mathbf{u}(k) \\ \mathbf{y}(k) &= \mathbf{C}\mathbf{x}(k) + \mathbf{D}\mathbf{u}(k) \end{align*} x(k+1)y(k)​=Ax(k)+Bu(k)=Cx(k)+Du(k)​

其中：
x(k)∈Rn\mathbf{x}(k) \in \mathbb{R}^nx(k)∈Rn 是 kkk 时刻的状态向量
u(k)∈Rm\mathbf{u}(k) \in \mathbb{R}^mu(k)∈Rm 是 kkk 时刻的输入向量
y(k)∈Rp\mathbf{y}(k) \in \mathbb{R}^py(k)∈Rp 是 kkk 时刻的输出向量
A∈Rn×n\mathbf{A} \in \mathbb{R}^{n \times n}A∈Rn×n 是状态矩阵
B∈Rn×m\mathbf{B} \in \mathbb{R}^{n \times m}B∈Rn×m 是输入矩阵
C∈Rp×n\mathbf{C} \in \mathbb{R}^{p \times n}C∈Rp×n 是输出矩阵
D∈Rp×m\mathbf{D} \in \mathbb{R}^{p \times m}D∈Rp×m 是直接传递矩阵

可见，实现的关键是矩阵的运算。
当系统结构复杂时，矩阵的阶数随之增加，
此时用CPU这种基于串行运算的处理器很可能无法实现实时运算，
因此通常是使用FPGA（能并行地对大规模矩阵进行运算）作为HIL设备的处理核心。

实际HIL设备仍然配有CPU，用以分担一部分计算任务，并和上位机进行交互。


那么剩下的问题便是两个：

  • 求取被控对象的离散状态方程。
  • 将离散状态方程转换为HDL代码。


在simulink中生成HDL代码

先解决上述问题的第二个。
simulink提供了HDL Coder用以将“合适的模块”搭建的模型转换为HDL代码。
打开simulink库浏览器，其中HDL Coder库在的模块便能进行此转换。

搭建的模型必须是离散的。

在simulink上方工作栏的建模>模型设置中可找到HDL Code Generation选项，
根据需要调整里面的配置，以实现生成代码的优化。
可参考官方文档


模型生成HDL代码的流程可使用HDL Workflow Advisor指导。
在simulink中，右键点击想要转换的子系统，找到HDL Code > HDL Workflow Advisor，
即可打开该工具。

HDL Workflow Advisor

可参考官方文档

将simscape模型转换为离散状态方程

simscape库的基本元件搭建的模型可以使用特定工具求得离散状态方程，
并且还能进一步生成基于HDL Coder库搭建的simulink模型，
再使用HDL Workflow Advisor即可。

simscape库是基于物理的，
或者说其用以运算与传输的信号是带有量纲的（伏特、安培、焦耳等），也因此才能转换为基于HDL Coder库搭建的simulink模型
因此不能与普通simulink器件连接（需要经过转换）

在simscape > Electrical > Specialized Power Systems中的器件是并不是基于物理的，因此也不能用于HDL转换。

通常带有彩色的simscape库元件（如蓝色、绿色、棕色等）的元件构成的模型才能用于转换，而黑色的simscape库元件并不能。

但注意一些复杂的颜色库器件也不能转换为基于HDL Coder库搭建的simulink模型。



普通库元件要将信号传输给simscape彩色库元件，可使用simulink-PS Conveter;
simscape彩色库元件要将信号传输给普通库元件，可使用PS-simulink Conveter;

使用simscape Electrical库搭建模型

使用该库搭建电路必须至少有一个电位参考点Electrial Reference。
使用simscape彩色库搭建的模型必须有且只有一个Solver Configration。

Solver Configration

可见官方文档

这里简要说明其中一些选项的选择。

  1. 对于最终要导出hdl代码的情况，勾选use local solver的选项，solver type选择后向欧拉即可。
    （如果solver type选择partitioning，能够提高仿真的速度，但不知为何使用sschdladvisor转换模型时耗时会大为增加。相关信息可看官方文档1与官方文档2）
    （关于求解器的说明可看此官方文档）

  2. 不要勾选Start simulation from steady state选项。

  3. 勾选use fixed-cost runtime consistency iterations，对于Nonlinear iterations，可通过simscape.getLocalSolverFixedCostInfo('modelname')获得，见官方文档

  4. 不要勾选Compute impulses

其余选项保持默认即可。

生成HDL Implementation Model

可参考官方文档

sschdladvisor函数能将simscape模型转换为离散状态方程，再输出，基于HDL Coder库搭建的模型（称为HDL Implementation Model）。

即可打开向导页面。

非线性元件不能转换，使用simscape.findNonlinearBlocks(‘modelname’)可找出模型中的非线性元件。

在向导页面中，State-space conversion一节是对simscape模型提取为状态空间方程并离散化。

需要注意的是，在提取状态空间方程时，要注意设置Simulation stop time及仿真结束时间。
此时间不能太小，因为工具会对Simcape模型中的所有模式进行仿真，提取对应的参数矩阵，如果时间太小，则没有充足的时间仿真完所有可能的模式。
同时也不能太大，如果太大，则工具会误以为仍有模式需要仿真，导致方程提取时间过长甚至无法停止（即stop time不能设置为inf）。
